# Scripts

Рабочие скрипты, прилагающиеся к книге.

## `zbx_audit_now.py`

**Что делает.** Подключается к Zabbix API через токен, выгружает в CSV:

- Все хосты с их группами, шаблонами, тегами, интерфейсами и состоянием
- Хосты с отсутствующими обязательными тегами
- Хосты с `unknown` значениями в ключевых тегах
- Хосты без шаблонов
- Хосты с недоступными интерфейсами
- Все активные проблемы с тегами и severity

**Зачем.** Главный инструмент Discovery (см. [глава 7](../docs/07_implementation_roadmap.md)).
За одну ночь — полная картина инвентаря с точки зрения текущего мониторинга.

**Требования.**

- Python 3.9+
- `pip install requests`
- Zabbix 6.0+ с API-токеном (Bearer auth)

**Запуск.**

```bash
export ZBX_URL="https://zabbix.example.local"
export ZBX_TOKEN="<your_api_token>"
python zbx_audit_now.py
```

На выходе в текущей директории:

```
01_hosts_all.csv                          — все хосты
02_hosts_missing_tags.csv                 — без обязательных тегов
03_hosts_unknown_owner_service_os.csv     — с unknown в ключевых тегах
04_hosts_no_templates.csv                 — без шаблонов
05_hosts_unavailable_interfaces.csv       — с проблемами интерфейсов
06_problems_active.csv                    — все активные проблемы
```

CSV в кодировке UTF-8 с BOM (`utf-8-sig`) — корректно открывается в Excel.

**Статус.** ⚫ **Lab-tested** — запускалось на Zabbix 6.4 c ~200 хостами.

**Важно.** Токен **никогда не хранится в коде**. Всегда через `os.environ`.
Учётка для токена должна быть **read-only** (отдельная Zabbix user group с view-only правами).

**Что добавить в следующих версиях:**

- Поддержка пагинации для больших инсталляций (>10К хостов)
- Опциональный фильтр по группам / тегам
- Выгрузка истории проблем за период (не только активные)
- Сравнение двух запусков (delta-режим — что изменилось)
