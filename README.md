# Zabbix Enterprise Guide

Как организовать мониторинг крупного предприятия — 1С, Exchange, MSSQL, SCADA, сетевое железо — и не утонуть в alert fatigue. Будь то legacy Zabbix, развёрнутый кем-то «как получилось», или проектирование с нуля.

Не про установку Zabbix step-by-step (см. [официальную документацию](https://www.zabbix.com/documentation)). Не про готовые шаблоны под импорт.

Про **инженерию эксплуатации**: теги, severity, runbook'и, SLA, разделение IT и OT, дашборды и эскалации.

---

## Для кого

- DevOps / системные инженеры, которым достался Zabbix "как есть" с предыдущей эпохи
- Архитекторы, проектирующие мониторинг для производственного предприятия с КИИ
- Тимлиды, переводящие команду от "красных лампочек" к зрелой эксплуатации
- Студенты и стажёры — для понимания, как устроен мониторинг в крупных компаниях

## Чего здесь нет

- Установки Zabbix step-by-step — это есть в [официальной документации](https://www.zabbix.com/documentation)
- Импортируемых production-ready шаблонов — здесь **требования и дизайн**, не имплементация
- Универсальных рецептов — везде есть оговорки про контекст

## Содержание

Главы 1–5 (принципы): сервис vs хост, severity как действие, теги и группы, LLD, слоистый дизайн.
Главы 6–8 (архитектура и внедрение): развёртывание, 90-day roadmap, implementation playbook, GitOps.
Главы 9–15 (эксплуатация и финал): runbooks, SLA, дашборды, операционка, шаблоны, антипаттерны.

Полное оглавление с описанием каждой главы — на [GitHub Pages](https://slowdownyw.github.io/zabbix-enterprise-guide/contents/).

У каждой главы в шапке указан статус — от 🟢 conceptually stable до 🔴 requirements only.

## Как контрибьютить

См. [`CONTRIBUTING.md`](./CONTRIBUTING.md).

Pull request'ы приветствуются. Особенно ценны:

- **Перевалидация шаблонов в проде** — если запускали что-то из главы 13 и оно сработало (или нет), напишите в Issue
- **Реальные runbook'ы** — в обезличенном виде
- **Опечатки и фактические ошибки** — без церемоний, через PR

При сабмите PR — желательно с указанием на каком Zabbix-релизе тестировали (6.x / 7.x).
