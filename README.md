# Zabbix Enterprise Guide

![Zabbix](https://img.shields.io/badge/Zabbix-6.x%20%2F%207.x-red?style=flat-square)
![License](https://img.shields.io/badge/docs-CC%20BY--SA%204.0-blue?style=flat-square)
![Chapters](https://img.shields.io/badge/глав-15-orange?style=flat-square)
![Status](https://img.shields.io/badge/статус-alpha-orange?style=flat-square)

**Практическое руководство по зрелому мониторингу крупной инфраструктуры** — 1С, Exchange, MSSQL, SCADA, сетевое оборудование, runbooks, SLA, дашборды и эксплуатационная модель.

Достался Zabbix «как есть» от предыдущей команды? Проектируете мониторинг для производства с КИИ с нуля? Команда тонет в усталости от алертов (alert fatigue) и игнорирует уведомления?

Эта книга — про **инженерию эксплуатации**: теги, severity как действие, runbook, SLA, разделение IT и OT, дашборды и эскалации. Не про установку по шагам (для этого есть [официальная документация](https://www.zabbix.com/documentation)).

📖 **[Читать онлайн → zabbix-enterprise-guide на GitHub Pages](https://slowdownyw.github.io/zabbix-enterprise-guide/)**

## 📦 Статус: alpha

Книга достигла альфа-версии: основной костяк готов, концепции устойчивы, материал можно использовать в реальных проектах с учётом дисклеймеров глав.

- **Замечания, опечатки и фактические ошибки** — заведите [GitHub Issue](https://github.com/slowdownyw/zabbix-enterprise-guide/issues/new) с шаблоном *Factual error*.
- **Запрос на новую главу или раздел** — отдельный issue с пометкой *Documentation improvement*.
- **Готовые правки текста** — Pull Request (см. [CONTRIBUTING.md](./CONTRIBUTING.md)).
- **Результат валидации шаблона в lab/prod** — issue *Template validation result* с указанием версии Zabbix.

## Что внутри

| Главы | Тема |
|-------|------|
| 1–5 | Принципы: сервис vs хост, severity, теги и группы, LLD, многослойный дизайн |
| 6–8, 16 | Архитектура и внедрение: развёртывание, Proxy Groups, 90-дневная дорожная карта, руководство по внедрению, GitOps |
| 9–12 | Эксплуатация: runbooks, SLA, дашборды, операционная модель |
| 13 | Шаблоны: требования к шаблонам |
| 15 | Завершение: антипаттерны |

У каждой главы в шапке статус — от 🟢 *conceptually stable* до 🔴 *requirements only*.

В [`examples/`](./examples/) — tag schema, trigger patterns, UserParameters, RACI, decision log, ADR-примеры.

## Для кого

- **DevOps / системные инженеры** — получили Zabbix «в наследство» и хотят привести его в порядок

## Как контрибьютить

PR и Issues приветствуются. Особенно ценны:

- **Перевалидация в проде** — запускали что-то из главы 13 и оно сработало (или нет)? Заведите Issue.
- **Реальные runbook** — в обезличенном виде.
- **Опечатки и фактические ошибки** — без церемоний, через PR.
- **Запросы на новые главы / разделы** — Issue с обоснованием и контекстом.

При отправке PR указывайте версию Zabbix (6.x / 7.x), на которой тестировали.

→ [CONTRIBUTING.md](./CONTRIBUTING.md)
